# Installation Guide

This document describes how to start a local Kong Gateway environment with the
GDCR custom plugin and an optional Redis instance using Docker Compose.[web:71]

---

## Prerequisites

- Docker 20.10 or later  
- Docker Compose 2.0 or later  
- `curl` or `httpie` for testing HTTP calls[web:70]

---

## 1. Docker Compose Setup

Create a file named `docker-compose.yml` in the root of this module and
paste the content below.

```yaml
version: '3.9'

services:
  kong-database:
    image: postgres:13
    environment:
      POSTGRES_USER: kong
      POSTGRES_DB: kong
      POSTGRES_PASSWORD: kongpass
    ports:
      - "5432:5432"
    volumes:
      - kong-db:/var/lib/postgresql/data

  kong-migrations:
    image: kong:3.4
    command: kong migrations bootstrap
    environment:
      KONG_DATABASE: postgres
      KONG_PG_HOST: kong-database
      KONG_PG_USER: kong
      KONG_PG_PASSWORD: kongpass
    depends_on:
      - kong-database

  kong-gateway:
    image: kong:3.4
    environment:
      KONG_DATABASE: postgres
      KONG_PG_HOST: kong-database
      KONG_PG_USER: kong
      KONG_PG_PASSWORD: kongpass
      KONG_PROXY_ACCESS_LOG: /dev/stdout
      KONG_ADMIN_ACCESS_LOG: /dev/stdout
      KONG_PROXY_ERROR_LOG: /dev/stderr
      KONG_ADMIN_ERROR_LOG: /dev/stderr
      KONG_ADMIN_LISTEN: 0.0.0.0:8001
      KONG_PLUGINS: bundled,gdcr
    ports:
      - "8000:8000"
      - "8001:8001"
      - "8443:8443"
      - "8444:8444"
    depends_on:
      - kong-migrations
    volumes:
      # Mount GDCR plugin code into Kong
      - ./plugin:/usr/local/share/lua/5.1/kong/plugins/gdcr

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data

volumes:
  kong-db:
  redis-data:
```

This file starts:

- PostgreSQL database for Kong
- Kong migrations container
- Kong Gateway with the gdcr custom plugin mounted from ./plugin

Redis 7 (optional backend for the Redis handler)

2. Start the Environment
From the same directory where docker-compose.yml is located, run:
```text
docker compose up -d
```
Check that Kong is up:
```text
curl http://localhost:8001/
```
You should receive a JSON response with Kong version and configuration.

3. Install the GDCR Plugin
Make sure your plugin code is available under ./plugin:

```text
plugin/
  handler-lua-local.lua
  handler-redis.lua
  schema.lua
  ...
```

The kong-gateway service mounts this folder into:

```text
/usr/local/share/lua/5.1/kong/plugins/gdcr
```
Kong is configured with:

```text
KONG_PLUGINS=bundled,gdcr
```
so the gdcr plugin becomes available as a standard Kong plugin.

4. Basic Test
   
Create a simple Service and Route, then attach the gdcr plugin.

Example using curl:

```text
# Create a mock upstream service
curl -X POST http://localhost:8001/services \
  --data name=gdcr-test \
  --data url=http://httpbin.org/anything

# Create a route that matches the GDCR pattern
curl -X POST http://localhost:8001/services/gdcr-test/routes \
  --data name=gdcr-route \
  --data paths[]=/sales/orders/create/salesforce

# Enable the gdcr plugin on this service
curl -X POST http://localhost:8001/services/gdcr-test/plugins \
  --data name=gdcr
```

Call the route:

```text
curl -i http://localhost:8000/sales/orders/create/salesforce
```
You should receive a JSON response generated by the GDCR handler (mock mode)
with interface_id, protocol and the generated_url.

5. Stopping the Environment
To stop and remove the containers while keeping the volumes:

```text
docker compose down
```
To remove containers and volumes:

```text
docker compose down -v
```
Notes:

-For Lua in‑memory tests, use handler-lua-local.lua as the plugin entry file.
For Redis stress tests, use handler-redis.lua and load the routes into the redis container before running load tests.
